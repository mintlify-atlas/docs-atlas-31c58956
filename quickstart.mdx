This guide will walk you through setting up Nanolytics and tracking your first events.

<Steps>
  <Step title="Install the package">
    First, install Nanolytics using your preferred package manager:

    ```bash
    npm install nanolytics
    ```

    <Info>
      Make sure you have Node.js >= 20.0.0 installed.
    </Info>
  </Step>

  <Step title="Initialize with a submission function">
    Create a submission function that will handle sending events to your analytics server. This function receives user information and an array of events, and should return a Promise.

    ```javascript
    import { init } from 'nanolytics';

    // Define your submission function
    const submitAnalytics = async (userInfo, events) => {
      const response = await fetch('/api/analytics', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ state: userInfo, data: events })
      });
      return response.json();
    };

    // Initialize Nanolytics
    init(submitAnalytics, { maxEventsLimitUntilSubmit: 150 });
    ```

    <Note>
      By default, Nanolytics will automatically trigger submission after 100 events. You can customize this with the `maxEventsLimitUntilSubmit` option.
    </Note>

    The `submitAnalytics` function receives:
    - `userInfo`: An object containing `language`, `platform`, and `userAgent` from the browser
    - `events`: An array of event objects with event name (`e`), timestamp (`t`), optional context (`c`), and any additional data

    <Tip>
      A successful response (200 OK) will cause the library to flush its localStorage store.
    </Tip>
  </Step>

  <Step title="Record some events">
    Use the `record()` function to track user interactions and events throughout your application:

    ```javascript
    import { record } from 'nanolytics';

    // Record a simple event
    record('ButtonClicked');

    // Record an event with context
    record('FormatByRule', 'ruleName', { from: 'Freewrite' });

    // Track form submission with additional context
    record('FormSubmitted', 'SignUpForm', { referrer: 'Homepage' });

    // Track user interactions
    const handleButtonClick = () => {
      record('ButtonClicked', 'HomePage');
    };
    ```

    Each event is automatically timestamped and stored in memory. Events accumulate in localStorage and will be submitted when:
    - The number of events exceeds your configured threshold (default: 100)
    - The user closes the page or switches tabs
    - You manually call the `commit()` function
  </Step>

  <Step title="See it in action">
    Nanolytics automatically handles session tracking and event persistence for you:

    ```javascript
    import { init, record, commit } from 'nanolytics';

    // Initialize once at app startup
    const submitAnalytics = async (userInfo, events) => {
      const response = await fetch('/api/analytics', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ state: userInfo, data: events })
      });
      return response.json();
    };

    init(submitAnalytics, { maxEventsLimitUntilSubmit: 150 });

    // The library automatically records a 'StartSession' event
    // It also tracks visibility changes (Hidden/Visible events)

    // Track user actions throughout your app
    record('StartSession');
    record('ButtonClicked', 'HomePage');
    record('FormSubmitted', 'SignUpForm', { referrer: 'Homepage' });

    // Optionally commit events before the user leaves
    window.onbeforeunload = () => {
      commit(); // Saves events to localStorage
    };
    ```

    <Warning>
      Events are stored in localStorage under the key `'events'`. Make sure this key doesn't conflict with other parts of your application.
    </Warning>

    <Tip>
      The library automatically sets up `window.onbeforeunload` to call `commit()`, so events are saved even if the user closes the page unexpectedly.
    </Tip>
  </Step>
</Steps>

## What happens next?

1. Events are stored in memory as you call `record()`
2. When the threshold is reached, your `submitAnalytics` function is called automatically
3. On successful submission (200 OK), events are cleared from localStorage
4. When the user leaves the page, remaining events are committed to localStorage
5. On the next visit, if saved events exceed the threshold, they're submitted automatically after a 3-second delay

## Advanced usage

You can use any HTTP client for submissions. Here's an example with axios:

```javascript
import axios from 'axios';
import { init } from 'nanolytics';

const submitAnalytics = async (userInfo, events) => {
  try {
    const response = await axios.put('/analytics-endpoint', {
      state: userInfo,
      data: events
    });
    return response.data;
  } catch (error) {
    console.error('Analytics submission failed:', error);
  }
};

init(submitAnalytics);
```
